# `test/` — controle de qualidade (TDD)

Testes que validam o código em `src/`. No ciclo `plan → run → test`, o `plan-dev` escreve os testes
que **falham** primeiro; o `run-dev` implementa até passarem; o `test-ops` valida de forma
independente. Rode-os em loop até o terminal aprovar a lógica.

Este README é só um marcador da pasta; substitua-o pelos testes reais do projeto.
